34 /* Matrix macros -------------------------------------------------------------*/
35 /* The following macros are used to allow for the creation of statically
36  allocated matricies. The matrix representation provide here consists
37  of three variables for each matrix. They are as follows:
38 
39  VariableName_data : The data for the given matrix
40  VariableName_ptr : A array of pointers to the first element in each row
41  VariableName : A pointer to the VariableName_ptr. In other words a
42  pointer to the array of pointers of the column vectors
43  of the array.
44 
45  Example: Lets say you want to create a 3x3 matrix. You could do this using
46  the following three lines of code:
47 
48  float A_data[9] = {0};
49  float A_ptr[3] = {&A_data[0], &A_data[3], &A_data[6]};
50  float **A = A_ptr;
51 
52  Now if you want to use any of the elements in the array you can do so as
53  you normally would if it was a standard two-dimensional array.
54 
55  // Example - set the diagonal elements to one
56  A[0][0] = 1.0f;
57  A[1][1] = 1.0f;
58  A[2][2] = 1.0f;
59 
60  You might be asking why not just initialize the array as a standard two
61  dimensional array such as:
62 
63  float A[3][3];
64 
65  The reason for this is well explained in page 20 of the book titled
66  "Numerical Recipes in C - The Art of Scientific Computing" Second
67  Edition. In a nutshell the standard two dimensional arrays of C are
68  not efficient for numerical methods because of the way in which C
69  references the array indicies. Take for example the following line
70  of code:
71 
72  A[i][j] = 1.0f;
73 
74  When this line is compiled to assembly it will result in something similar
75  to the following line of code:
76 
77  *(A + 3*i + j) = 1.0f;
78 
79  This means "to the address of A add 3 times i plus j and return
80  the value thus addressed. This code results in two integer adds and one
81  integer multiplication for every array index reference. An alternative
82  method that is more efficient is to have a pointer to an array of
83  pointers given as **A. This will result in code similar to:
84 
85  *(*(A+i) + j) = 1.0f;
86 
87  This means "to the address of A add i, take the value thus addressed as
88  a new address, add j to it, return the value addressed by this new
89  address". In this version only two integer adds are required, which
90  eliminates a integer multiplication. This will result in more
91  efficient embedded code. The penalty paid for this more efficient method
92  is that it will require the storage of additional terms in the form of a
93  array of pointers to the rows of the matrix. This is a slight
94  inconvenience as it will require that the user to initialize three
95  variables as opposed to one for each matrix. The macros provided here
96  are designed to simplify this process. A matrix can be initialized using
97  these macros as shown below:
98 
99  CreateMatrix(A, 3, 3, {1, 0, 0, 0, 1, 0, 0, 0, 1});
100 
101  This line will result in the folloing lines of code:
102 
103  static float A_data[9] = {1, 0, 0, 0, 1, 0, 0, 0, 1};
104  static float *A_ptr[3] = {&A_data[0], &A_data[3], &A_data[6]};
105  static float **A = A_ptr;
106 
107  The macros given below will only work for matricies with sizes less
108  than 12 in each dimension. These macros are only compatible with C99 and
109  above versions of C. If you are using strict ANSI C then you will need to
110  write out the three lines shown above for each matrix you want to
111  initialize.
112  */
